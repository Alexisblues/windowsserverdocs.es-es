# <a name="deploy-remote-access-in-a-multi-forest-environment"></a>Implementar Acceso remoto en un entorno con varios bosques

>Se aplica a: Windows Server (canal semianual), Windows Server 2016

Las herramientas de configuración de acceso remoto (consola de administración de acceso remoto y los cmdlets de Windows PowerShell) están diseñadas para funcionar bien en un entorno de bosque único que contiene uno o varios dominios. Sin embargo, cuando Acceso remoto se implementa en un entorno con varios bosques, el administrador de Acceso remoto deberá realizar parte de la configuración manualmente para que la implementación sea correcta. En esta guía se detallan los pasos de planeación y configuración de una implementación con varios bosques, incluido el caso en el que se usa la autenticación mediante contraseña de un solo uso (OTP).  
  
-   [Planear una implementación de varios bosques](Plan-a-Multi-Forest-Deployment.md)  
  
-   [Configurar una implementación con varios bosques](Configure-a-Multi-Forest-Deployment.md)
