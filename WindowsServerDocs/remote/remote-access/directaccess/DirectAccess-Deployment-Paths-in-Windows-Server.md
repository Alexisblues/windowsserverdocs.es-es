# <a name="directaccess-deployment-paths-in-windows-server"></a>Rutas de acceso de implementación de DirectAccess en Windows Server

>Se aplica a: Windows Server (canal semianual), Windows Server 2016

En este tema se proporciona una lista de la documentación de las dos rutas de implementación principales de acceso remoto: **Básico** y **avanzada**.  
  
Puede usar la siguiente sección para obtener una descripción de las diferencias entre las rutas de acceso de implementación avanzada y básica de DirectAccess, y puede usar los vínculos de documentación para localizar a la Guía de implementación que se adapta mejor a sus objetivos.  
  
## <a name="deploy-basic-directaccess"></a>Implementación de DirectAccess básica  
Con una implementación básica de DirectAccess, DirectAccess se configura con la configuración predeterminada mediante un asistente, sin necesidad de establecer la configuración de infraestructura, como una entidad de certificación (CA) o grupos de seguridad de Active Directory.  
  
-   [Implementar un único servidor de DirectAccess mediante el Asistente para introducción](../../remote-access/directaccess/single-server-wizard/Deploy-a-Single-DirectAccess-Server-Using-the-Getting-Started-Wizard.md). Puede usar a esta guía para implementar DirectAccess básica en un entorno de producción.  
  
## <a name="deploy-advanced-directaccess"></a>Implementación de DirectAccess avanzada  
Con una implementación avanzada de DirectAccess, implementar un único servidor de DirectAccess y configurar los servidores de infraestructura de red para admitir DirectAccess.  
  
-   [Implementar un único servidor de DirectAccess con configuración avanzada](../../remote-access/directaccess/single-server-advanced/Deploy-a-Single-DirectAccess-Server-with-Advanced-Settings.md). Puede usar a esta guía para implementar DirectAccess con configuración avanzada en un entorno de producción.
