Estas notas resumen los problemas más graves en el sistema operativo Windows Server, como las maneras de evitar o solucionar los problemas conocidos. Para obtener información sobre las nuevas características en esta versión, consulte [What ' s New in Windows Server versión 1803](whats-new-in-windows-server-1803.md). Desproteger [contenedores de Windows sobre](https://docs.microsoft.com/virtualization/windowscontainers/about/) si está interesado en ejecutar un servidor de Windows, versión 1803, el contenedor.

Este artículo se actualiza continuamente. Si se identifican los problemas conocidos, se deberá documéntelos aquí. 


## <a name="software-defined-datacenter"></a>Centro de datos definido por software

Características del centro de datos definido por el software, como espacios de almacenamiento directo, definidas por software de red y blindadas máquinas virtuales no están incluidas en Windows Server, versión 1803. Como se describe en [actualización del canal semianual de Windows Server](https://cloudblogs.microsoft.com/windowsserver/2018/03/29/windows-server-semi-annual-channel-update/), el canal semianual de Windows Server se centra en los contenedores y los escenarios de aplicación que se benefician de una innovación más rápida. 

Si necesita roles de infraestructura, use una versión de canal de servicio a largo plazo: Windows Server 2016 (ya está disponible) o [Windows Server 2019](https://cloudblogs.microsoft.com/windowsserver/2018/03/20/introducing-windows-server-2019-now-available-in-preview) (que se lanzarán este año).

Estamos comprometidos a crear la mejor plataforma para infraestructura hiperconvergente y seguimos desarrollar nuevas características y mejorar las existentes según sus comentarios. Agradecimiento por ayudarnos a llegar a [más de 10.000 clústeres de espacios de almacenamiento directo](https://blogs.technet.microsoft.com/filecab/2018/03/27/storage-spaces-direct-momentum)! Estamos deseando al uso compartido más adelante este año.
